Common non-standard request headers[edit]

Field name Description Example
X-Requested-With mainly used to identify Ajax requests. Most JavaScript frameworks send this header with value of XMLHttpRequest X-Requested-With: XMLHttpRequest
DNT[5] Requests a web application to disable their tracking of a user. This is Mozilla's version of the X-Do-Not-Track header (since Firefox 4.0 Beta 11). Safari and IE9 also have support for this header.[6] On March 7, 2011, a draft proposal was submitted to IETF.[7] The W3C Tracking Protection Working Group is producing a specification.[8] DNT: 1 (Do Not Track Enabled)

DNT: 0 (Do Not Track Disabled)
X-Forwarded-For[9] a de facto standard for identifying the originating IP address of a client connecting to a web server through an HTTP proxy or load balancer X-Forwarded-For: client1, proxy1, proxy2

X-Forwarded-For: 129.78.138.66, 129.78.64.103
X-Forwarded-Proto[10] a de facto standard for identifying the originating protocol of an HTTP request, since a reverse proxy (load balancer) may communicate with a web server using HTTP even if the request to the reverse proxy is HTTPS X-Forwarded-Proto: https
Front-End-Https[11] Non-standard header used by Microsoft applications and load-balancers Front-End-Https: on
X-ATT-DeviceId[12] Allows easier parsing of the MakeModel/Firmware that is usually found in the User-Agent String of AT&T Devices X-Att-Deviceid: GT-P7320/P7320XXLPG
X-Wap-Profile[13] Links to an XML file on the Internet with a full description and details about the device currently connecting. In the example to the right is an XML file for an AT&T Samsung Galaxy S2. x-wap-profile: http://wap.samsungmobile.com/uaprof/SGH-I777.xml
Proxy-Connection[14] Implemented as a misunderstanding of the HTTP specifications. Common because of mistakes in implementations of early HTTP versions. Has exactly the same functionality as standard Connection header. Proxy-Connection: keep-alive
